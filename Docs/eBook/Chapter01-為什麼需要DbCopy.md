# 為什麼需要 DbCopy？

## 資料庫遷移的痛點

在軟體開發的日常工作中，資料庫遷移與同步是最常見卻也最令人頭痛的任務之一。無論是：

- **開發環境同步**：將正式環境的資料庫結構複製到本地開發環境
- **測試環境準備**：為 QA 團隊建立一份與生產環境結構一致的測試資料庫
- **跨伺服器遷移**：將舊伺服器的資料庫搬移到新伺服器
- **Schema 版本對齊**：確保多個環境的資料庫結構保持一致

這些工作往往需要 DBA 或資深開發者手動操作，過程繁瑣且容易出錯。

### 常見的困境

1. **物件繁多**：一個中型資料庫可能包含數百張資料表、數十個檢視、上百個預存程序，手動逐一比對幾乎不可能。

2. **相依性複雜**：資料表之間有外鍵關聯，預存程序參考檢視，函數使用自訂型別——物件之間的相依性形成了一張複雜的有向圖，複製順序稍有不對就會失敗。

3. **資料量龐大**：動輒數百萬筆的資料表，傳統的 INSERT 語句效率低落，需要專門的大量載入工具。

4. **人為錯誤**：手動執行 DDL 腳本時，漏掉一個索引、忘記建立一個自訂型別，往往要到應用程式報錯時才會發現。

## 現有工具的不足

市面上並非沒有資料庫比較與同步工具，例如：

| 工具 | 限制 |
|------|------|
| SQL Server Management Studio (SSMS) | 僅支援 SQL Server，且結構比較功能有限 |
| pgAdmin | 僅支援 PostgreSQL，無自動同步功能 |
| Redgate SQL Compare | 商業授權費用高昂，僅支援 SQL Server |
| dbForge Schema Compare | 商業軟體，單一資料庫引擎 |
| Liquibase / Flyway | 專注於版本控制遷移，非即時比較同步 |

這些工具各有其價值，但在以下場景往往力有未逮：

- 需要**跨資料庫引擎**的統一介面（同時處理 SQL Server 和 PostgreSQL）
- 需要**輕量化**的部署（一個執行檔即可運作，無需安裝）
- 需要**分階段控制**的同步（先結構、再資料、再索引、最後外鍵）
- 需要**Web 介面**方便遠端操作（不需要安裝桌面軟體）

## DbCopy 的設計目標

DbCopy 的誕生源自於一個簡單的需求：**我需要一個工具，能夠快速比較兩個資料庫的差異，並將缺少的物件複製過去。**

基於這個核心需求，我們定義了以下設計目標：

### 目標一：完整的物件覆蓋

不只是資料表，還要支援：
- 資料表（Tables）
- 檢視（Views）
- 預存程序（Stored Procedures）
- 函數（Functions）
- 使用者定義型別（User-Defined Types）
- 使用者定義資料表型別（User-Defined Table Types）
- 序列（Sequences）

### 目標二：智慧的相依性分析

系統必須自動分析物件之間的相依關係，告知使用者「這個檢視依賴哪些資料表」、「這個預存程序使用了哪些自訂型別」。

### 目標三：分階段的複製控制

資料表的複製分為四個階段：
1. **結構**（Schema）：建立資料表定義
2. **資料**（Data）：大量複製資料列
3. **索引**（Indexes）：建立非主鍵索引
4. **外鍵**（Foreign Keys）：建立外部參照

使用者可以選擇一次執行全部階段，或逐階段手動控制。

### 目標四：零安裝部署

透過 .NET 的 Single File Publishing 技術，將整個應用程式（包含 Web 伺服器、靜態檔案、所有依賴套件）打包成一個單一執行檔。使用者只需要下載、雙擊執行，瀏覽器就會自動開啟。

### 目標五：跨平台支援

同一套程式碼可以建置為五個平台的執行檔：
- Windows x64
- Linux x64
- Linux ARM64
- macOS x64（Intel）
- macOS ARM64（Apple Silicon）

## 本書適合的讀者

本書適合以下背景的讀者：

- **中高階 .NET 開發者**：已有 C# 基礎，想學習如何設計完整的工具型應用程式
- **資料庫管理員（DBA）**：想了解資料庫物件的程式化管理方式
- **系統架構師**：想參考介面導向設計、分層架構、跨資料庫抽象的實務做法
- **全端開發者**：想學習 ASP.NET Core + Razor Pages + JavaScript 的整合方式

本書假設讀者已具備以下基礎知識：
- C# 語法基礎
- SQL 基礎（SELECT、CREATE TABLE 等）
- HTTP 與 REST API 的基本概念
- HTML/CSS/JavaScript 的基本知識

不需要事先了解的內容（本書會從頭講解）：
- ASP.NET Core 的中介軟體管線
- Dapper ORM 的使用方式
- SQL Server 與 PostgreSQL 的系統目錄
- 大量資料載入（Bulk Copy）技術
- Single File Publishing 的原理
